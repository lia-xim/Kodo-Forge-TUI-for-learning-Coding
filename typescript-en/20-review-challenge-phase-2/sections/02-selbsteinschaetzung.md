# Self-Assessment: 20 Questions

> Answer each question honestly with: Confident / Unsure / No Idea
>
> For "Unsure" or "No Idea" → revisit the corresponding lesson.

---

## Type Narrowing (L11-L12)

1. Can I use typeof, instanceof and the in operator for type narrowing?
2. Can I write custom type predicates (is-functions)?
3. Can I implement discriminated unions with exhaustive checks?
4. Do I understand the difference between a type guard and a type assertion?

## Generics (L13-L14)

5. Can I write generic functions and interfaces with constraints?
6. Can I implement the Builder Pattern with generic type growth?
7. Can I make pipe() type-safe with overloads?
8. Do I understand the difference between generics and any/unknown?

## Utility Types (L15)

9. Can I correctly use Partial, Required, Readonly, Pick, Omit, Record?
10. Can I build custom utility types like DeepPartial or PartialBy?
11. Do I understand how utility types work internally with mapped types?

## Mapped Types (L16)

12. Can I read and write the `{ [K in keyof T]: ... }` syntax?
13. Can I use key remapping with the as clause?
14. Can I deliberately apply and remove modifiers (+/-, readonly, ?)?

## Conditional Types (L17)

15. Can I write `T extends U ? X : Y` and nested conditionals?
16. Can I use infer to extract types from patterns?
17. Do I understand distributive conditional types and can I control distribution?

## Template Literal Types (L18)

18. Can I use template literal types for string patterns and event names?
19. Can I do string parsing with infer in template literals?

## Modules & Declarations (L19)

20. Can I write declaration files and augment modules?
